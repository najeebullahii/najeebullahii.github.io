FinQuery — Natural Language Analytics for E-Commerce

An AI-powered business intelligence app that lets anyone ask plain English questions about a real e-commerce database and get instant, visualised answers — no SQL knowledge required.

Overview

What This Project Does

In most companies, getting answers from a database requires either knowing SQL yourself or waiting for an analyst to write a query. FinQuery removes that barrier entirely. A business user, product manager, or executive can type a question like "Which product categories generate the most revenue?" and get a chart and data table back within seconds.

The system sends the question to Google's Gemini 2.5 Flash AI model along with a detailed description of the database structure. Gemini interprets the intent and writes a precise SQL query. That query runs against a SQLite database containing the full Olist Brazilian E-Commerce dataset, and results come back as both an interactive chart and an explorable data table.

The App in Action

The main interface uses a chat layout. You type a question, the system processes it, and the response appears as a chart followed by the data table. A collapsible panel beneath each response shows the exact SQL query Gemini generated.

Asking "What are the top 10 product categories by total revenue?" — the system joined four tables and aggregated pricing data across 112,650 order line items.
Every response includes the SQL query Gemini wrote. The queries are production-quality, using proper table aliases, JOIN conditions, GROUP BY, and ORDER BY clauses.

Technical Design

How It Was Built

The application is structured across four Python files, each with a specific responsibility.

database_setup.py — Handles one-time ingestion of nine CSV files into a local SQLite database. Cleans column names, fills nulls, converts datetime columns to strings for SQLite compatibility, and loads each table using SQLAlchemy.
nl_to_sql.py — The core AI logic. Constructs a detailed prompt including the full schema — table names, column types, foreign key relationships, and explicit rules — then sends it to the Gemini API with the user's question. Strips any markdown formatting before returning clean SQL.
app.py — The Streamlit front end. Manages conversation history using session state, auto-generates bar charts or scatter plots based on the result shape, and includes a processing lock to prevent simultaneous API calls.
Security: Two independent layers. First, the prompt instructs Gemini to write only SELECT statements. Second, the SQLite connection is opened with mode=ro, which means the database driver rejects any write operation regardless of what SQL is submitted.
